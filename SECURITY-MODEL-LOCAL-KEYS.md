# Freedom Trader 安全模型说明

## 核心结论

**私钥不上传服务器。**
**私钥只在本地浏览器插件后台解密和使用。**
**普通网页拿不到私钥明文。**

## 这意味着什么

Freedom Trader 的签名和发送交易逻辑运行在浏览器插件自己的后台环境中，而不是运行在你正在浏览的网站页面里。

所以：

- 网站页面本身不能直接读取你的私钥
- 网站页面不能直接拿到私钥明文
- 插件不会把私钥上传到我们的中心化服务器保存或托管
- 我们的后端不替用户保管私钥，也不替用户代签

## 私钥在哪里

私钥只存在于两个地方：

1. **本地加密存储**
   - 私钥会先在本地加密后再保存
   - 使用的是 `PBKDF2 + AES-256-GCM`

2. **本地插件后台内存**
   - 用户解锁后，私钥只会在本地插件后台短暂解密
   - 用于签名和发送交易
   - 普通前端 UI 页面拿不到这部分明文

## 不经过中心化托管

Freedom Trader 的安全模型不是“把私钥交给平台保管”。

而是：

- **不托管私钥**
- **不上传私钥到服务器**
- **只在用户本地插件后台使用私钥**

这可以避免一类非常典型的中心化风险：

**后端持有私钥，一旦服务器泄漏，用户资产一起暴露。**

## 浏览器插件层的隔离

Freedom Trader 当前的关键签名逻辑在插件后台 `background service worker` 中运行。

这意味着：

- 网站页面 JavaScript 和插件后台不是同一个执行上下文
- 网站页面不能直接访问插件后台中的私钥变量
- 前端界面层也不会直接暴露私钥明文

## 对外可以直接说的话

如果你要对外发布，可以直接使用下面这句：

**Freedom Trader 不上传、不托管用户私钥。私钥只在本地浏览器插件后台解密和使用，普通网页拿不到私钥明文。**

## 更短版本
