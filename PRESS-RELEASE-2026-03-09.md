# Freedom Trader 2.1.0 版本更新通讯稿

Freedom Trader 今日正式发布 2.1.0 版本。

本次更新是一轮围绕 **安全性、执行质量、路由能力** 展开的重要升级。新版本在延续 BSC + Solana 双链聚合交易能力的基础上，进一步强化了本地签名安全模型，并完成了 BSC 路由的整体升级。

本次版本最值得强调的一点是：

**Freedom Trader 不上传、不托管用户私钥。私钥只在本地浏览器插件后台解密和使用，普通网页拿不到私钥明文。**

这意味着，Freedom Trader 的安全模型并不是“把私钥交给中心化后端保管”，而是尽可能把签名与交易执行留在用户本地环境中完成。对于交易工具来说，这是一条非常明确的安全边界，也是在当前环境下最值得被强调的产品原则之一。

在 BSC 侧，Freedom Trader 已完成 FreedomRouter v6 路由升级，统一支持：

- Four.meme
- Flap
- PancakeSwap

新版本可以自动识别不同协议与阶段状态，在更统一的交易入口下完成买卖、报价与授权目标判定，减少因路径差异带来的复杂度和错误概率。

与此同时，本次升级也进一步强化了交易执行层的安全与稳定性，包括：

- BSC 交易统一增加 `deadline` 时效保护
- 授权目标由链上合约返回，避免前端本地猜测
- FreedomRouter Proxy 与 Implementation 已完成 BscScan 验证，链上逻辑可直接审阅

当前 BSC 主网 FreedomRouter 地址为：

`0x444444444444147c48E01D3669260E33d8b33c93`

除了路由升级，Freedom Trader 也继续保持原有双链能力，包括 Solana 侧 Pump.fun / PumpSwap 路径支持、多钱包批量交易、Jito 加速确认、自定义 RPC 与侧边栏交易体验。

Freedom Trader 2.1.0 不是一次单纯叠加功能的更新，而是一次更明确回答“用户资产安全边界在哪里”的版本升级。

对外而言，我们更希望用户记住这句话：

**你的私钥不经过我们的服务器。**
**你的私钥只在你自己的本地插件后台使用。**

这就是 Freedom Trader 2.1.0 最核心的升级价值。
