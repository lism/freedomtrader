# Freedom Trader 2.1.0 更新公告

## 推特短版

Freedom Trader 2.1.0 已上线。

这次不是小修小补，是一次以 **安全性 + 路由能力** 为核心的大版本更新：

- 私钥 **不上传服务器**
- 私钥只在 **本地插件后台** 解密和使用
- 前端页面 **拿不到私钥明文**
- BSC 路由升级到 **FreedomRouter v6**
- 新增 **Flap** 支持
- 合约已完成 **BscScan 验证**

BSC 新主网地址：
`0x444444444444147c48E01D3669260E33d8b33c93`

这意味着：
**不存在“后端托管私钥被拖库”这一类中心化托管风险。**

## 推特正式版

Freedom Trader 2.1.0 已发布。

本次更新重点不是单纯增加功能，而是把“安全模型”彻底讲清楚并落到实现上：

**1. 私钥不上传服务器**
Freedom Trader 不托管你的私钥，不把私钥发到我们的后端，也不依赖中心化服务器替你签名。

**2. 私钥只在本地插件后台使用**
私钥只会在浏览器插件本地的 `background service worker` 中短暂解密，用于签名和发送交易。

**3. 前端页面拿不到私钥明文**
普通 UI 页面不接触私钥明文，降低前端界面层误暴露风险。

**4. 本地加密 + 自动锁定**
私钥在本地用 `PBKDF2 + AES-256-GCM` 加密存储，并支持自动锁定。

**5. BSC 路由升级**
FreedomRouter v6 已上线，统一支持：
- Four.meme
- Flap
- PancakeSwap

**6. 可审计**
当前 BSC Proxy 与 Impl 都已完成 BscScan 验证，链上逻辑可直接查看。

新 BSC 主网 Proxy：
`0x444444444444147c48E01D3669260E33d8b33c93`

## 更强势但不绝对的版本

Freedom Trader 2.1.0 上线。

我们最想强调的不是功能，而是安全模型：

**你的私钥不经过我们的服务器。**
**你的私钥只在本地插件后台使用。**
**我们不托管你的私钥。**

这不能等于“绝对没有任何风险”，但它确实避开了最典型、最致命的一类风险：
**中心化后端持有私钥，单点泄漏导致整批用户资产暴露。**

同时本次 BSC 路由升级到 FreedomRouter v6，支持 Four.meme / Flap / PancakeSwap，合约已开源可审计。

## 用词建议

如果你要发推，建议用这些说法：

- `私钥不上传服务器`
- `只在本地插件后台使用`
- `前端页面拿不到明文`
- `不托管私钥`
- `链上逻辑可审计`

不建议直接写这些绝对化表述：

- `绝对不可能被盗`
- `100%安全`
- `永远不会出事`

更稳妥也更专业的说法是：

**我们从架构上避免“后端托管私钥”这类中心化单点风险。**
